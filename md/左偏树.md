    如果我们想要让两个堆合并，暴力的做法就需要 O(nlogn) 的时间复杂度，显然这还不是最优解。那么为了应对合并的情况，可并堆就诞生了，它可以用 $O(logn+logm)$ 的时间复杂度完美胜任合并两个堆的任务。这是竞赛中常见的一种实现方法：

### 左偏树

    「对于一棵二叉树，我们定义 外节点 为左儿子或右儿子为空的节点，定义一个外节点的 dist 为 $1$，一个不是外节点的节点 dist 为其到子树中最近的外节点的距离加一。空节点的 dist 为 $0$ 。」

    设当前结点为 p，则 distp​ 为子树中深度最小的空结点的到 p 的距离

![](https://cdn.luogu.com.cn/upload/image_hosting/pbr0140f.png)

3,9,10 号结点的 dist 都是 1, 而 2,6 号结点的 dist 为 2.

### 左偏树的定义

    每个结点的左儿子 dist 都不小于它右儿子的 dist，如下：

![](https://cdn.luogu.com.cn/upload/image_hosting/ofwei3xm.png)

    显然，一个结点的 dist 能且只能被它两个子结点中 dist 小的更新 ，而右儿子的 dist 永远比左儿子小，所以可以直接使用右儿子的 dist 来更新当前结点的 dist。

得到如下公式:

$distp​=distr​+1$

### 合并操作（merge）

1. 把根结点权值小的堆（根为 x）作为基础，它的所有左儿子不做出改变，而把另一个堆（根为 y）和 x 的右儿子继续递归合并。
2. 递归到 x,y 中的一个为空结点时结束，返回不为空的结点编号
3. 更新 dist 值,且把不满足左偏性质的左右儿子互换。

   ![](https://cdn.luogu.com.cn/upload/image_hosting/p3wtvxy4.png)

   总代码

```cpp
#include <iostream>
using namespace std;
int n,m,fa[100010],del[100010],lf[100010],rt[100010],dist[100010];
struct node{
    int v,id;
}a[100010];
int find(int x){
    return x==fa[x]?x:fa[x]=find(fa[x]);
}
//合并
int merge(int x,int y){//两个堆顶
    if(!x||!y)return x+y;
    if(a[x].v>a[y].v)swap(x,y);
    rt[x]=merge(rt[x],y);
    if(dist[lf[x]]<dist[rt[x]])swap(lf[x],rt[x]);
    dist[x]=dist[rt[x]]+1;
    return x;
}
//
int main(){
    cin>>n>>m;
    for(int i = 1;i<=n;i++){
        cin>>a[i].v;//一开始有n个小根堆
        a[i].id = i;
        fa[i]=i;
    }
    for(int i = 1;i<=m;i++){
        int op,x,y;cin>>op;
//op=1 将第x与第y个数所在的小根堆合并
//op=2 输出第x个数所在堆的最小数，并将其删除
        if(op==1){
            cin>>x>>y;
            if(x==y||del[x]||del[y])continue;
            x = find(x);y = find(y);
            fa[x]=fa[y]=merge(x,y);
        }else{
            cin>>x;
            if(del[x]){
                cout<<-1<<'\n';continue;
            }x = find(x);
            cout<<a[x].v<<'\n';
            del[x]=1;
            fa[lf[x]]=fa[rt[x]]=fa[x]=merge(lf[x],rt[x]);
            lf[x]=rt[x]=dist[x]=0;
        }
    }
}
```

### 删除任意数

1. 给定权值
   
       其实包括普通的二叉堆和左偏树，都是不太支持删除指定权值的数的。因为需要遍历整个堆来寻找，O(nk) 的时间复杂度对于竞赛无法接受。如果实在想实现这种操作，可以在外面再敲一个平衡树之类的数据结构维护结点编号和权值的关系，或者可以通过 map 维护，但毕竟不是平衡树的题目，局限性还是有的。不过不必担心，这种 ~~双层大数据结构的毒瘤~~ 题目应该不会在正式竞赛中出现。

2. 给定结点编号
   
   这种要求就十分好满足了，有两种情况。
   
   - 删除的是根结点，和上文中的 pop函数 一样，不再赘述。
   - 删除的是某个结点的子结点，这时其实仍然可以通过合并左右儿子的方式来删除，但问题是，这样删除会影响到这个结点的祖先。所以我们对每个节点记录它的父结点，每次合并后往上查找是否有需要更新的值，有则更新（注意这时不能再使用路径压缩了）。
     
     

### 修改某个堆中全部结点的权值

    可以给根结点打懒惰标记，只有查询或合并的时候才更改结点的权值，而懒惰标记的形式与线段树类似。

    在合并操作中也需下传。
